# 配置真相源

这个 starter 的配置面现在收口到一组 canonical replacement surfaces。

改这里，才是在改实际网站；不要再维护一份运行时相邻的镜像配置。

## 当前网站真正读取的主配置

| 范围 | 主配置 | 说明 |
| --- | --- | --- |
| 单站品牌、联系方式、社交链接、默认 SEO、公司事实 | `src/config/single-site.ts` | 当前单站主真相。 |
| sitemap、robots、lastmod、允许进入 sitemap 的公开页面清单 | `src/config/single-site-seo.ts` | SEO 公开页面和搜索引擎抓取规则的主配置。 |
| 导航 href | `src/config/single-site-links.ts` 和 `src/config/single-site-navigation.ts` | 页面链接从主路径配置派生。 |
| 页面表达输入 | `src/config/single-site-page-expression.ts` | 首页表达、产品/市场按钮/行动入口、custom project support 等页面输入面。 |
| 产品市场和 catalog truth | `src/config/single-site-product-catalog.ts` 与 `src/constants/product-specs/**` | 产品事实不能只改一个产品卡片文件。 |
| 运行时环境变量 | `src/lib/env.ts` | 网站运行时读取的 server/client 环境变量配置。 |

## 同步规则

这些值不应无理由漂移：

- 公司名
- 站点名
- 默认域名或 base URL
- 联系邮箱
- 主社交链接
- SEO title template 的品牌部分

这些值可以在不同层表达不同角度，但必须有说明：

- 电话格式：当前网站真正读取的主配置可以保留 starter placeholder，但 client launch 前必须替换为负责人确认过的联系方式。
- founded year / established year：如果只是 demo placeholder，不能被当作真实业务事实。
- 页面表达：`src/config/single-site-page-expression.ts` 是运行时输入面，覆盖首页表达、产品/市场按钮/行动入口、custom project support 等页面输入。
- 产品事实：`src/config/single-site-product-catalog.ts` 与 `src/constants/product-specs/**` 要一起看，不能只改一个字段就当作产品事实已改完。

## 派生项目替换顺序

1. 先改 `src/config/single-site.ts`。
2. 品牌、域名、SEO 或搜索引擎抓取变更时，检查 `src/config/single-site-seo.ts`。
3. 导航变更时，检查 `src/config/single-site-navigation.ts` 和 `src/config/single-site-links.ts`。
4. 页面表达、首页顺序、产品/市场按钮/行动入口或 custom project support 变更时，检查 `src/config/single-site-page-expression.ts`。
5. 产品事实变更时，检查 `src/config/single-site-product-catalog.ts` 和 `src/constants/product-specs/**`。
6. 再替换 `content/pages/{locale}/*.mdx`。
7. 最后检查 `messages/{locale}/critical.json`、`messages/{locale}/deferred.json` 和 `public/images/**`。

## 后续治理规则

- 不要再新增一份 runtime-adjacent mirror config。
- 如果人工同步继续造成实际漂移，再用单独计划评估是否需要结构合并。
- 不要为了绕过当前真相源临时引入新的 `defineSiteDefinition(...)`。
