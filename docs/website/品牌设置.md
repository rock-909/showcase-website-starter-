# 品牌设置

品牌信息的 canonical authoring source 是 `src/config/single-site.ts`，不要把公司名、域名、邮箱散落到组件里。

`src/config/website/profile.ts` 和 `src/config/website/seo.ts` 是兼容镜像层；如果项目还保留这两个文件，它们必须和 `src/config/single-site.ts` 同步，不能当主来源。

## 必填项

- `src/config/single-site.ts`：公司身份、联系方式、社交链接、默认 SEO、公司事实
- `src/config/website/profile.ts`：兼容镜像层里的公司身份
- `src/config/website/seo.ts`：兼容镜像层里的默认标题、描述、OG 信息
- `navigation.ts`：导航结构
- `contact.ts`：询盘表单接收方式和联系信息

## 原则

页面组件只读取配置和翻译，不直接写死品牌事实。

如果新项目需要改品牌，优先改 `src/config/single-site.ts`，并同步检查 `src/config/website/profile.ts` / `src/config/website/seo.ts` 这类镜像层；其次改 `content/pages/**`，最后才改组件。

## 导航响应式规则

本 starter 和从它派生出的网站采用同一套导航断点：

```text
< 840px       移动菜单
840-1199px   紧凑桌面导航
>= 1200px    完整桌面导航
```

含义：

- `< 840px`：只显示 Logo 与菜单按钮，语言切换和 CTA 放进菜单。
- `840-1199px`：显示主导航，并把 Contact 快捷入口和紧凑语言切换放在右侧；品牌文字可能保持截断，避免挤乱头部。
- `>= 1200px`：显示完整品牌、主导航、语言切换和 Contact / 联系快捷入口。

不要按设备名硬套断点。真正标准是：主导航应该尽早可见，但头部不能因为品牌名、语言按钮或 CTA 被挤乱。
