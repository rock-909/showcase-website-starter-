# 品牌设置

品牌信息的当前网站真正读取的主配置是 `src/config/single-site.ts`，不要把公司名、域名、邮箱散落到组件里。

更完整的配置分层说明见 [配置真相源](配置真相源.md)。这份文档区分“当前网站真正读取的主配置”、“给新项目替换时参考的辅助配置”和“兼容用的镜像配置”。

先改主配置，再同步检查辅助配置和镜像配置。不要以为改了 `src/config/website/*`，就一定改了实际网站。

## 运行时主配置

- `src/config/single-site.ts`：公司身份、联系方式、社交链接、默认 SEO、公司事实。
- `src/config/single-site-seo.ts`：sitemap、robots、lastmod、允许进入 sitemap 的公开页面清单。品牌、域名、SEO、搜索引擎抓取变更都要检查这里。
- `src/config/single-site-navigation.ts` 和 `src/config/single-site-links.ts`：运行时导航和 href 来源。导航变更不能只看 `src/config/website/navigation.ts`。
- `src/config/single-site-page-expression.ts`：运行时 page expression 输入面，覆盖首页表达、产品/市场按钮/行动入口、custom project support 等页面输入。

## 需要同步检查的兼容用镜像配置

这里的镜像配置，也可以理解为兼容用的镜像层：它要跟主配置对齐，但不能替代主配置。

- `src/config/website/profile.ts`：兼容用的镜像配置里的公司身份，必须和 `src/config/single-site.ts` 的意图保持一致，但不能覆盖主配置。
- `src/config/website/seo.ts`：兼容用的镜像配置里的默认标题、描述、OG 信息，必须和 `src/config/single-site.ts` / `src/config/single-site-seo.ts` 的意图保持一致。

## 给新项目替换时参考的辅助配置，不是运行时主来源

- `src/config/website/navigation.ts`：可帮助派生项目理解导航结构，但实际运行导航要看 `src/config/single-site-navigation.ts` 和 `src/config/single-site-links.ts`。
- `src/config/website/contact.ts`：可说明联系和询盘替换面，但真实表单流向还要看网站运行时读取的环境变量、API route 和 lead pipeline。
- `src/config/website/homepage.ts`：当前只是给新项目替换时参考的辅助配置；运行时页面组合和按钮/行动入口主要看 `src/config/single-site-page-expression.ts`。
- `src/config/website/products.ts`：只代表入口卡片，不代表完整产品 catalog。不要只改这个文件就以为产品事实改完了。

## 页面内容与法务内容

- `content/pages/{locale}/*.mdx`：页面正文、法务页、可替换的长内容。
- `messages/{locale}/*.json`：界面文案和多语言短文案。
- `public/images/**`：品牌图片、产品图、图标和其他静态素材。

## 原则

页面组件只读取配置和翻译，不直接写死品牌事实。

如果新项目需要改品牌，优先改 `src/config/single-site.ts`。如果涉及品牌、域名、SEO 或搜索引擎抓取，再检查 `src/config/single-site-seo.ts`；如果涉及导航，再检查 `src/config/single-site-navigation.ts` 和 `src/config/single-site-links.ts`；如果涉及首页表达、产品/市场按钮/行动入口或 custom project support，再检查 `src/config/single-site-page-expression.ts`。

这些主配置确认后，再同步检查 `src/config/website/profile.ts` / `src/config/website/seo.ts` 这类兼容用的镜像配置，以及 `src/config/website/navigation.ts` / `contact.ts` / `homepage.ts` / `products.ts` 这类辅助配置。它们是给 starter 派生和 AI 工作流看的替换面，不是实际网站的同级主来源。

最后再改 `content/pages/**`、`messages/**` 和 `public/images/**`。不要为了省事直接改组件，也不要在 Phase 1 新增 `defineSiteDefinition(...)` 或提前退役镜像配置；这些要放到单独计划里做。

## 导航响应式规则

本 starter 和从它派生出的网站采用同一套导航断点：

```text
< 840px       移动菜单
840-1199px   紧凑桌面导航
>= 1200px    完整桌面导航
```

含义：

- `< 840px`：只显示 Logo 与菜单按钮，语言切换和按钮/行动入口放进菜单。
- `840-1199px`：显示主导航，并把 Contact 快捷入口和紧凑语言切换放在右侧；品牌文字可能保持截断，避免挤乱头部。
- `>= 1200px`：显示完整品牌、主导航、语言切换和 Contact / 联系快捷入口。

不要按设备名硬套断点。真正标准是：主导航应该尽早可见，但头部不能因为品牌名、语言按钮或按钮/行动入口被挤乱。
